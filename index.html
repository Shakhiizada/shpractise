QUESTIONS&ANSWERS:

Что такое git workflow?
Git workflow идет как подпроцесс управления в Git, который показывает где должна быть определенные изменения и команды.

Как работает git workflow
Git workflow позволяет команде хорошо работать над проектом, разделяя задачи и обеспечивая легкость в интеграции изменений,позволяет изменения. легко вносить правки.Также программа может использоваться для управления документами, контроля за исполнением поручений, а также других операций

Для чего нужен git workflow?
Git workflow нужен для структурированности и Упрощение сотрудничества

Что такое git rebase? Отличия от git merge?
Это способ в Git,благодоря которому можно объединить изменение двуx веток .В отличие от операции слияния (merge) Rebase переписывает историю коммитов.

Что такое git fetch?Отличия от git pull
Это способ загрузки коммитов, файлы и ссылки из удаленного репозитория в свой локальный репозиторий.
Команда git fetch загружает удаленное содержимое, но не изменяет состояние локального репозитория,a как git pull загружает удаленное содержимое и сразу пытается изменить состояние локального репозитория, чтобы оно соответствовало этому содержимому

Что такое GitHub Actions
GitHub Actions это способ благодаря которому можно автомотизировать процессы разработки прямов репозитории на GitHub
